Data Format

The import/export format for frequency databases is an extended version of CHIRP's CSV format. (Why that? Because it was the first documented and straightforward existing data format I found.) Here is CHIRP's documentation of the column formats. Not all columns are currently used by ShinySDR; in particular, those related to transmitter settings are not.

The minimum columns required for a useful database are Frequency (in megahertz) and Name; for example, 

Frequency,Name
10,WWV

The extensions are as follows:

  • The Frequency column may contain two frequencies separated by a hyphen “-”. In this case, the record defines a band instead of a channel. The lower frequency should always be specified first. (TODO: Make sure code either rejects or fixes this.)

  • Two additional columns Latitude and Longitude can be used to specify the geographic location of a specific station. Each value, if not blank, should be a decimal number of degrees (negative values are south or west) and, if you want to be exact, in the WGS 84 coordinate system.
